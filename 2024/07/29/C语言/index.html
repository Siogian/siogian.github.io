程序结构和构建过程

C 程序包含一个或多个源文件，以及相关的头文件。一个源文件连同其通过 #include 预处理指令直接或间接包含的所有头文件一起被称为预处理翻译单元。经预处理之后的预处理翻译单元被称为翻译单元。翻译单元之间可能存在互相引用（如外部函数引用，外部全局变量引用等），每个翻译单元被独立的编译成目标文件，最后链接到一起形成可执行文件。将源文件转换到可执行文件整个的过程，称为构建。

中文 English
源文件 source files
头文件 headers
预处理翻译单元 preprocessing translation unit
翻译单元 translation unit
目标代码 object code
可执行程序 preprocessing
编译 compiling
汇编 assembling
链接 linking
构建 build

以上术语在不同地方可能有代替表达，扩大或缩小解释。如源代码表示程序员使用编程语言编写的原始代码，即源文件中的内容；保存目标代码的文件称为目标文件，构建有时候又称翻译；编译可能会表示预处理和编译，甚至可能直接指整个构建过程，大部分工具链会将翻译单元先翻译成汇编代码，在由汇编器汇编成目标代码，在这种情况下，编译仅指从翻译单元翻译为汇编代码的过程。

在 Windows 上安装 GCC

  1. 通过 MSYS2 安装 MinGW-w64 和 GCC。

  2. 使用 PowerShell 将 MinGW-w64 bin 文件夹的路径 <MYSYS2 Installation Folder>\ucrt64\bin 添加到 PATH 环境变量。

    [!NOTE]
    如果你没有修改 MSYS2 的默认安装目录，<MYSYS2 Installation Folder> 应为 C:\msys64。

  3. 重新启动 PowerShell，以使 PATH 环境变量生效。
